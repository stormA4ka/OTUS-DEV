### СОЗДАЕМ ВИРТУАЛЬНОЕ ОКРУЖЕНИЕ

Чтобы создать виртуальное окружение в папке с проектом, выполните следующие шаги:

### 1. Откройте терминал или командную строку
Перейдите в папку с вашим проектом. Например:

```bash
cd C:\Users\dergilev\Desktop\projects\OTUS-DEV\lesson.03
```

### 2. Создайте виртуальное окружение
Используйте команду `python -m venv`, чтобы создать виртуальное окружение. Вы можете назвать его как угодно, но обычно используют имя `venv`.

```bash
python -m venv venv
```

2.1. **Использование конкретной версии Python:**

   Если у вас установлена версия Python 3.11, и вы хотите создать виртуальное окружение с этой версией, вы можете использовать полный путь к исполняемому файлу Python 3.11:

   ```bash
   C:\Users\dergilev\AppData\Local\Programs\Python\Python311\python.exe -m venv venv
   ```

2.2. **Использование версии Python, установленной через Anaconda:**

   Если у вас установлен Anaconda, и вы хотите использовать конкретную версию Python, установленную через Anaconda, вы можете использовать полный путь к исполняемому файлу Python из Anaconda:

   ```bash
   C:\Users\dergilev\Anaconda3\python.exe -m venv venv
   ```

2.3. **Использование версии Python, установленной через pyenv (на Linux/macOS):**

   Если вы используете `pyenv` для управления версиями Python, вы можете указать нужную версию следующим образом:

   ```bash
   pyenv exec python3.11 -m venv venv
   ```


Эта команда создаст папку `venv` в вашем проекте, где будут храниться все файлы виртуального окружения.

### 3. Активируйте виртуальное окружение
Теперь вам нужно активировать виртуальное окружение. Команда активации зависит от операционной системы:

- **Windows (cmd) :**

  ```bash
  venv\Scripts\activate
  ```

- **macOS/Linux:**

  ```bash
  source venv/bin/activate
  ```

или так:

- **PowerShell:**
```bash
.\venv\Scripts\Activate.ps1
```
После создания виртуального окружения, вы можете активировать его и проверить версию Python:

Далее выполняем:
```bash
python --version
```
Это должно вывести версию Python, которую вы указали при создании виртуального окружения.


После активации виртуального окружения, в командной строке появится префикс `(venv)`, указывающий на то, что вы работаете в виртуальном окружении.

### 4. Установите необходимые пакеты
Теперь вы можете устанавливать необходимые пакеты в виртуальное окружение, используя `pip`. Например:

```bash
pip install requests
```
### 4.1 Или установить из файла зависимостей

```bash
(venv) PS C:\Users\dergilev\Desktop\projects\OTUS-DEV\lesson.03> pip install -r requirements.txt
```

### 5. Деактивируйте виртуальное окружение
Когда вы закончите работу, вы можете деактивировать виртуальное окружение, выполнив команду:

```bash
deactivate
```
Теперь вы вернетесь в глобальное окружение Python.

### 6. Добавьте `.gitignore` (опционально)
Если вы используете Git, рекомендуется добавить папку `venv` в `.gitignore`, чтобы она не попала в репозиторий. Создайте файл `.gitignore` в корне проекта и добавьте в него следующую строку:

```
venv/
```

Теперь ваше виртуальное окружение готово к использованию!

При создании виртуального окружения с помощью команды `python -m venv venv`, используется версия Python, которая связана с исполняемым файлом `python`, который вы вызываете. Если у вас установлено несколько версий Python, вы можете указать нужную версию, используя полный путь к исполняемому файлу Python.


### Команды:


### РАБОТАЕМ С DOCKERFILE

### Шаг 1: Создайте Dockerfile

Создайте файл `Dockerfile` в директории `C:\Users\dergilev\Desktop\projects\OTUS-DEV\lesson.03` и добавьте следующий код:

```Dockerfile
# Dockerfile для FastAPI приложения
# Для сборки образа выполните:
# docker build -t myfastapiapp .
# Для запуска контейнера выполните:
# docker run -d -p 8000:8000 myfastapiapp

# Используем базовый образ Python 3.9 slim
FROM python:3.9-slim

# Устанавливаем рабочую директорию внутри контейнера
WORKDIR /app

# Копируем файл requirements.txt в рабочую директорию
COPY requirements.txt .

# Устанавливаем зависимости
RUN pip install --no-cache-dir -r requirements.txt

# Копируем все файлы и папки приложения в рабочую директорию
COPY . .

# Указываем порт, который будет использоваться приложением
EXPOSE 8000

# Запускаем веб-сервер Uvicorn с параметрами host 0.0.0.0 и port 8000
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### Создаем и запускаем Dockerfile

1. **Комментарии**:
   - В начале файла добавлены комментарии с инструкциями по сборке и запуску контейнера.

2. **Базовый образ**:
   - Используется базовый образ `python:3.9-slim`, который содержит минимальный набор зависимостей для запуска Python.

3. **Рабочая директория**:
   - Устанавливается рабочая директория `/app` внутри контейнера.

4. **Копирование файла `requirements.txt`**:
   - Копируется файл `requirements.txt` в рабочую директорию.

5. **Установка зависимостей**:
   - Устанавливаются зависимости из файла `requirements.txt` с использованием `pip`. Опция `--no-cache-dir` предотвращает сохранение кэша пакетов, что уменьшает размер образа.

6. **Копирование всех файлов и папок приложения**:
   - Копируются все файлы и папки из текущей директории на хосте в рабочую директорию `/app` внутри контейнера.

7. **EXPOSE 8000**: 
   - Эта директива указывает, что контейнер будет использовать порт 8000 для связи. Она не открывает     порт на хосте, а просто документирует, какой порт будет использоваться.

8. **Запуск Uvicorn**:
   - Запускается веб-сервер Uvicorn с параметрами `--host 0.0.0.0` (чтобы сервер был доступен снаружи контейнера) и `--port 8000` (порт, на котором будет работать сервер).

### Шаг 2: Сборка Docker-образа

Перейдите в директорию `C:\Users\dergilev\Desktop\projects\OTUS-DEV\lesson.03` и выполните команду для сборки Docker-образа:

```bash
docker build -t myfastapiapp .
```

### Шаг 3: Запуск контейнера

После успешной сборки образа, запустите контейнер с помощью следующей команды:

```bash
docker run -d -p 8000:8000 myfastapiapp
```

- `-d`: Запуск контейнера в фоновом режиме (detached mode).
- `-p 8000:8000`: Проброс порта 8000 внутри контейнера на порт 8000 на хосте.
- `myfastapiapp`: Имя образа, который вы создали.

Теперь ваше FastAPI приложение должно быть доступно по адресу `http://localhost:8000`.
